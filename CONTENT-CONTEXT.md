## Guidelines for Writers

### When Concept Already Explained

**Options**:
1. **Brief Reference**: "As we discussed in [previous piece], good explanations are hard to vary..."
2. **Link Back**: "For a full explanation of hard-to-vary, see [link]"
3. **Quick Reminder**: "Remember: hard-to-vary means you can't swap details without losing explanatory power"
4. **Ask User**: "This requires explaining hard-to-vary. It's been covered before. Repeat it, reference it, or skip?"

### When to Repeat Anyway

- Concept is central to this piece (can't understand without it)
- Audience is likely different (e.g., LinkedIn vs YouTube)
- Months have passed since last explanation
- User explicitly requests repetition

### Default Approach

**Before explaining a concept that's been covered**:
1. Check this tracker
2. Ask user: "Hard-to-vary was explained in [piece]. Should I: a) Fully re-explain, b) Brief reminder, c) Just reference it?"
3. Proceed based on user preference

## Tone & Style Evolved

### What We've Learned

| Issue | Solution | Status |
|-------|----------|--------|
| Too preachy | More humble, questioning tone ("Consider this...", "What if...") | ✓ Implemented (human-creativity-ai-era) |
| Insufficient counterarguments | Dedicate sections to strongest objections, steelman them | ✓ Implemented (human-creativity-ai-era) |
| Repetitive explanations | Use context tracker | ✓ Implemented |

### Tone Best Practices (from human-creativity-ai-era, creativity-universal-root-ai-era, identity-as-compression)

**Exploratory language that works**:
- "Consider this..."
- "What if..."
- "Let me show you what I found"
- "Do you see the pattern?" ← **Especially effective** (creativity-universal-root-ai-era)
- "Here's what I've come to understand..." ← **Humble framing works well**
- "I don't know..." (when genuinely uncertain)
- "Try this right now" ← **Immediate action language** (identity-as-compression)

**Practical Utility Best Practices** (identity-as-compression):
- **Specificity formula**: Tool + Time + Numbered Steps + Example + Outcome
- Examples: "Open Google Doc" (tool), "Set 5-minute timer" (time), "Step 1, 2, 3" (numbered), "Example: Sarah..." (concrete), "You'll have 3 labels identified" (outcome)
- **Decision logic**: "If X, then do Y" rather than "reflect on patterns"
- **Calendar integration**: "Write it in your calendar RIGHT NOW" (mandatory, not suggested)
- **Measured outcomes**: Include 90-day results, 12-week transformations with specific numbers
- **Self-check methods**: Week 1/4/12 markers, common mistakes, troubleshooting
- **Zero vague advice**: Never "think about" or "consider" - always "open [tool] and complete [steps]"

**Avoid**:
- "You must understand"
- "Obviously" / "Clearly"
- "The truth is"
- Lecturing tone

### Counterarguments Addressed

1. **"AI IS doing explanatory work—it writes essays, solves novel problems, generates creative ideas"** → Distinguished novel outputs from explanatory reach. AI generates novel combinations but cannot create explanations with reach beyond training data or correct errors based on reality-testing.

2. **"Even if explanatory work is safe, I still need to eat TOMORROW—not in 10 years"** → Acknowledged as "strongest objection, honestly." Framework provides direction for strategic learning, recognition of commoditization, and mental clarity—doesn't eliminate problem but provides better navigation.

3. **"This just feels like moving the goalposts—anything AI can do, you call 'pattern'"** → Acknowledged as "fatal objection if true." Key response: Deutsch's pattern/explanation distinction was formulated BEFORE modern AI (2011), not ad-hoc redefinition. Falsifiable: if AI creates genuinely novel explanations with reach, distinction fails.

### Practical Utility Innovations

- **5-minute audit**: List 10 tasks with pattern/explanatory identification clues
- **5-day experiment workflow**: Day-by-day structure (Monday→identify, Tuesday→test, Wednesday→measure, Thursday→calculate, Friday→decide)
- **4-skill reallocation**: Problem formulation, strategic judgment, error correction, creative synthesis - each with brief explanation
- **4-question decision filter**: Pattern/explanation, retain value, formulate/execute, judgment under uncertainty
- **4-question monthly self-check**: Pattern tasks remaining, explanatory skills developed, time allocation shift, specific judgment example

### Key Mechanistic Insights

- **Paradox structure**: Two problems (economic + epistemological), not one unified phenomenon
- **Economic mechanism**: Pattern work being automated → real disruption, appropriate anxiety
- **Epistemological error**: Treating AI as competitor in intelligence vs tool for pattern automation
- **Resolution**: Shift from pattern mastery (depreciating) to explanatory creativity (appreciating)
- **Historical pattern**: Every pattern-automation technology increased demand for explanatory work
