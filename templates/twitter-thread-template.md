# Twitter/X Thread Template

## Thread Length
8-12 tweets (optimal)
Can extend to 15 for complex topics, but each tweet must earn its place

## Character Limit
280 characters per tweet (but leave room for retweets)

## Structure

---

### TWEET 1: THE HOOK

**Goal**: Stop the scroll. Make them want to read.

**Techniques**:
- Bold claim
- Surprising fact
- Provocative question
- Clear contradiction
- Promise of insight

**Format Options**:

**Option A - The Bold Claim**:
```
[Controversial but defensible statement]

[Why it matters in one line]

[Thread indicator: 🧵]
```

**Option B - The Question Hook**:
```
Why does [common phenomenon] happen?

The standard explanation is wrong. Here's what's really going on:

🧵
```

**Option C - The Paradox**:
```
[Thing A] and [Thing B] seem contradictory.

But both are true. The resolution reveals [insight]:

🧵
```

**Example**:
```
"Follow your passion" is bad career advice.

Not because it's wrong, but because it's what David Deutsch calls a "bad explanation."

Here's why that matters: 🧵
```

---

### TWEET 2: SET UP THE PROBLEM

**Goal**: Establish what needs explaining

**Format**:
```
[What people are confused about]

[Why it matters]

[Why current explanations fail]
```

**Example**:
```
The advice sounds good: find what you love, then do it for work.

But notice: you could replace "passion" with "calling" or "purpose" or "excitement" and the advice still "works."

That's the problem. Let me explain...
```

---

### TWEETS 3-5: THE BAD EXPLANATION(S)

**Goal**: Show why common explanations fail

**Format for each**:

**Tweet 3**:
```
Common explanation: [what people think]

[Why it seems plausible]

But here's the issue...
```

**Tweet 4**:
```
This explanation is "easy to vary":

- [Component that can change]
- [Another component that can change]
- [Result: explains nothing]
```

**Tweet 5**:
```
Why this matters:

Easy to vary = arbitrary details = no contact with reality

Good explanations are HARD to vary. You can't change them without breaking them.
```

**Example**:
```
Tweet 3:
People think: "Find your passion first, then build career around it"

Sounds reasonable. Lots of successful people are passionate about their work.

But watch what happens when we test this explanation...

Tweet 4:
You can swap "passion" for:
• Calling
• Purpose
• Excitement
• Interest

...and the advice still "works"

That means the core concept is arbitrary. It explains everything, therefore nothing.

Tweet 5:
This is what Deutsch calls an "easy-to-vary" explanation.

Like ancient myths: swap the gods, change the story, still "explains" seasons.

Bad explanations can fit any observation because they're not constrained by reality.
```

---

### TWEETS 6-8: THE GOOD EXPLANATION

**Goal**: Build the better explanation

**Tweet 6 - The Mechanism**:
```
Here's what actually happens:

[The mechanism, clearly stated]

[Why this is necessary, not arbitrary]
```

**Tweet 7 - The Evidence/Reach**:
```
This explains not just [original phenomenon], but also:

- [Related thing 1]
- [Related thing 2]
- [Related thing 3]

This is "reach" - good explanations explain MORE than intended.
```

**Tweet 8 - Why It's Hard to Vary**:
```
Notice: you CAN'T vary the components without breaking the explanation.

[Example of what breaks if you change it]

That's how you know it's a GOOD explanation.
```

**Example**:
```
Tweet 6:
What actually happens:

People develop passion THROUGH skill development and problem-solving.

Passion follows mastery, not the other way around.

Skill → Autonomy & Impact → Passion

Tweet 7:
This explains:

• Why "follow your passion" fails (cart before horse)
• Why skilled people love their work (mechanism is clear)
• Why passion changes (as skills/problems change)
• Why career switches work (learn new skills → develop new passion)

Tweet 8:
You can't swap "skill development" for "finding yourself" or "cosmic destiny."

Break the mechanism → explanation fails.

That constraint = contact with reality.

That's a GOOD explanation.
```

---

### TWEETS 9-10: IMPLICATIONS

**Goal**: Make it practical/actionable

**Tweet 9 - Theoretical Implication**:
```
This reveals something important:

[Broader insight about knowledge/explanations/problem-solving]
```

**Tweet 10 - Practical Implication**:
```
What this means for you:

• [Actionable insight 1]
• [Actionable insight 2]
• [Actionable insight 3]
```

**Example**:
```
Tweet 9:
This is why Deutsch's framework matters:

Good explanations vs bad explanations isn't academic philosophy.

It's the difference between advice that helps vs advice that misleads.

Tweet 10:
What this means for your career:

• Don't wait to "find" passion - build skills instead
• Pick problems worth solving, passion follows
• If not passionate YET, you may just need more mastery
• Career advice should specify mechanisms, not platitudes
```

---

### TWEET 11: THE CLOSE

**Goal**: Memorable ending + call to action

**Format Options**:

**Option A - The Summary**:
```
TL;DR:

[One sentence core insight]

[Why it matters]

[What to do]
```

**Option B - The Open Question**:
```
This raises an important question:

[New problem this creates]

[Invitation to think/discuss]
```

**Option C - The Call to Action**:
```
If this framework is new to you:

Read [Deutsch's book/article]

It changes how you think about [domain].

Thread ends. Thoughts?
```

**Example**:
```
The framework:

Bad explanations = easy to vary, no mechanism, no reach
Good explanations = hard to vary, clear mechanism, wide reach

Apply this to ANY domain.

Most popular advice fails the test. But good explanations exist—seek them out.
```

---

### OPTIONAL TWEET 12: THE LINK

If you have a related article/resource:

```
I wrote more about this here:

[Link]

Or follow me @[handle] for more on applying Deutsch's framework to practical problems.
```

## Thread-Writing Best Practices

### Each Tweet Should:
- [ ] Work standalone (somewhat)
- [ ] Flow naturally from previous tweet
- [ ] Advance the argument
- [ ] Stay under 280 chars (leave room for RT)
- [ ] Be clear and readable

### Use Formatting:
- **Bullets** for lists
- **Line breaks** for readability
- **Numbers** for sequences
- **Bold claims** at start of tweets
- **Questions** to maintain engagement

### Avoid:
- ❌ Dense blocks of text
- ❌ Jargon without explanation
- ❌ More than 2-3 points per tweet
- ❌ Weak transitions between tweets
- ❌ Boring middle tweets (maintain energy!)

## The Hook Checklist

Your first tweet should make people feel:
- [ ] Curiosity ("I need to know this")
- [ ] Surprise ("That can't be right...")
- [ ] Recognition ("I've wondered about this")
- [ ] Urgency ("This matters now")

Test: Would YOU click through?

## The Flow Checklist

Reading through the whole thread:
- [ ] Each tweet follows naturally from previous
- [ ] No logical leaps
- [ ] Maintains momentum
- [ ] No dead spots
- [ ] Builds to payoff

## Voice for Threads

### Do:
- Conversational and direct
- Punchy and energetic
- Clear and specific
- Confident but not arrogant

### Don't:
- Academic or stuffy
- Meandering or vague
- Preachy or condescending
- Unnecessarily complex

## Common Thread Patterns

### Pattern 1: Myth-Busting
1. Common belief
2. Why it seems right
3. Why it's actually wrong
4. What's really going on
5. Why this matters
6. What to do instead

### Pattern 2: Explanation Analysis
1. Phenomenon that needs explaining
2. Bad explanation #1
3. Why it fails
4. Bad explanation #2
5. Why it fails
6. Good explanation
7. Why it works
8. Implications

### Pattern 3: Problem-Solving
1. Problem stated
2. Why it matters
3. Why it seems hard
4. The key insight
5. How to apply it
6. What changes

### Pattern 4: Framework Application
1. Introduce framework concept
2. Show how it works
3. Apply to example 1
4. Apply to example 2
5. Apply to example 3
6. General principle
7. How to use it

## Length Calibration

### 8 tweets: Simple idea, clear application
- Quick setup
- One bad explanation
- One good explanation
- Clean takeaway

### 10 tweets: Standard thread
- Full setup
- Compare 2 explanations
- Show implications
- Actionable ending

### 12+ tweets: Complex topic
- More thorough setup
- Multiple bad explanations
- Detailed good explanation
- Rich implications
- BUT: Each tweet must earn its place

## Engagement Boosters

### Mid-Thread:
- Ask a question (tweet 5-6)
- Make bold claim (tweet 7-8)
- Unexpected insight (tweet 6-7)
- Relatable example (anywhere)

### End:
- Direct question to audience
- Controversial but defensible claim
- Practical application
- Resource link

## Testing Your Thread

Before posting, read it aloud:
- [ ] Does it sound natural?
- [ ] Are transitions smooth?
- [ ] Is energy maintained?
- [ ] Is payoff worth the read?
- [ ] Would you share this?

## Example Thread Arc

**Topic**: "Why software estimation is always wrong"

```
1. Hook: "Software estimates are always wrong. Always. This isn't because programmers are bad at math—it's because we're using a bad explanation of what programming is."

2. Setup: "We treat programming like construction: plan it out, estimate time per task, add it up. Seems logical. Every company does this. Why does it always fail?"

3. Bad Explanation: "The standard explanation: 'We just need better estimation techniques. More data. More experience. More padding.'"

4. Why It Fails: "But notice: you can swap in ANY technique (story points, t-shirt sizes, fibonacci numbers) and it still 'works' until it doesn't. Easy to vary = bad explanation."

5. The Real Issue: "Here's what's actually happening: Programming is knowledge creation, not brick-laying. You can't estimate when you'll solve a problem you don't understand yet."

6. The Mechanism: "Construction: Known unknowns, same task as before, predictable. Programming: Unknown unknowns, never the same problem twice, unpredictable by nature."

7. The Reach: "This explains: - Why experienced devs aren't better at estimates - Why padding doesn't help - Why Agile ditched estimation - Why research projects can't be planned"

8. It's Hard to Vary: "Can't replace 'knowledge creation' with 'construction' → explanation breaks. Can't remove 'unknown unknowns' → explanation breaks. Constrained by reality."

9. Implication: "So what? Stop trying to estimate the unestimatable. Instead: - Break into smaller pieces - Deliver incrementally - Adjust as you learn - Embrace uncertainty"

10. Close: "Deutsch's framework: Good explanations are hard to vary. Apply this to your domain. Most 'best practices' are just easy-to-vary myths. Find the real mechanisms."

11. Link: "More on knowledge creation vs. execution: [link] Follow for more Deutsch framework applications to practical problems."
```

## Remember

Threads are about:
- **Clarity** over cleverness
- **Insight** over information
- **Story** over structure
- **Value** over volume

Every tweet should make someone think "Yes, keep going" or "Aha!"

If a tweet doesn't do that, cut it or improve it.

The thread should feel inevitable—like each idea naturally leads to the next until you reach a satisfying conclusion.
