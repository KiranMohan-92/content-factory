## The Science: Why Threads Go Viral

### Algorithm Signals That Matter

| Signal | Multiplier | What It Means |
|--------|------------|---------------|
| Someone replies to your tweet | **13.5x** | Design for reply-ability |
| You reply to their reply | **75x** | Engage in first 2-3 hours |
| Profile click + stay >2 min | **10x** | Create author curiosity |
| Bookmark/Save | **High** | Create "saveable" content |
| Retweet | **1x** | Make tweets quotable |
| Like | **0.5x** | Least valuable |

### Critical Timing
- **First hour**: Early engagement determines if algorithm promotes
- **First 3 seconds**: Hook must stop the scroll
- **24-36 hours**: Natural decay unless viral (resets clock)

### The Viral Equation
```
Reach = (Hook Strength × Dwell Time × Reply Velocity) / Friction
```

## Thread Length

**Optimal**: 10-16 tweets
- Under 10: Not enough dwell time
- Over 16: Diminishing returns, reader fatigue
- Sweet spot: 12-14 tweets

**Each tweet**: Under 250 characters (leave room for RT, better mobile readability)

## Thread Architecture

```
[1] HOOK ─────────────────────────── Stop the scroll (50% of success)
[2-3] SETUP ──────────────────────── Relatable problem, stakes
[4-5] BAD EXPLANATIONS ───────────── What's wrong with common view
[6] CLIFFHANGER ──────────────────── "But here's what actually works..."
[7-10] GOOD EXPLANATION ──────────── Mechanism, examples, evidence
[11-12] PAYOFF ───────────────────── Actionable framework
[13] OBJECTION ───────────────────── Steelman counter + response
[14] CALLBACK ────────────────────── Loop back to hook
[15-16] CTA ──────────────────────── Save/Reply/Follow triggers
```

## TWEET 1: THE HOOK

### The 50% Rule
Your hook determines 50% of your thread's performance. Invest accordingly.

### Hook Formula
```
[Pattern Interrupt] + [Stakes] + [Promise] + [Thread Indicator]
```

### 5 Proven Hook Patterns

#### Pattern 1: The Contrarian Callout
Attack sacred cows directly.

```
"If you can't explain it simply, you don't understand it."

This is wrong.

And believing it is why you feel like a fraud.

Here's what's actually happening: 🧵
```

**Why it works**: Challenges famous quote → creates cognitive dissonance → must resolve by reading.

---

#### Pattern 2: The Specific Paradox
Two contradictory truths that demand explanation.

```
The best writers write less.
The best coders delete code.
The best thinkers forget ideas.

Paradox—until you see the mechanism: 🧵
```

**Why it works**: Multiple examples create pattern → paradox creates tension → mechanism promises resolution.

---

#### Pattern 3: The Hidden Experience
Name something they've felt but never articulated.

```
You've felt this:

You understand something deeply.

Someone asks you to explain.

You open your mouth and... nothing.

You're not stupid. You're hitting a fundamental limit: 🧵
```

**Why it works**: Immediate recognition ("that's me!") → validation → curiosity about explanation.

---

#### Pattern 4: The Specific Number
Numbers create instant credibility.

```
272 words.

That's all Lincoln needed to redefine American democracy.

10,000-word speeches from the same era? Forgotten.

The compression algorithm he used: 🧵
```

**Why it works**: Specificity = credibility → contrast creates stakes → "algorithm" promises actionable insight.

---

#### Pattern 5: The Percentage Claim
Quantified contrarian take.

```
90% of productivity advice makes you LESS productive.

Not because it's wrong—

Because it's what David Deutsch calls a "bad explanation."

Here's how to spot the difference: 🧵
```

**Why it works**: Specific percentage → counterintuitive claim → framework promises detection method.

---

### Hook Anti-Patterns (What Kills Threads)

| Bad Hook | Problem | Better Version |
|----------|---------|----------------|
| "Productivity is hard" | Vague, obvious | "90% of productivity advice fails" |
| "Here are 10 tips" | Generic, no tension | "The tip everyone shares is wrong" |
| "Thread on leadership" | No curiosity gap | "The leadership advice that backfires" |
| "I learned something" | Self-focused | "[Specific insight] changes everything" |

### Hook Checklist
- [ ] Stops scroll in 3 seconds
- [ ] Creates curiosity gap (FOMO)
- [ ] Specific, not vague
- [ ] Contrarian or counterintuitive
- [ ] Under 280 characters
- [ ] Has 🧵 indicator

## TWEETS 2-3: THE SETUP

### Goal
Make them feel seen. Establish stakes.

### Format
```
## Tweet 2: The Experience
[Describe the problem they recognize]

[Make them nod: "that's exactly how it feels"]

[Short lines. White space.]

## Tweet 3: The Stakes
[Why this matters]

[What they're losing by not understanding]

[Transition to "why current solutions fail"]
```

### Example
```
## Tweet 2
You understand something deeply.

Someone asks you to explain it.

You open your mouth and... word salad.

Their eyes glaze over.

You try again. Worse.

## Tweet 3
This isn't imposter syndrome.

You're not stupid.

You're solving one of the hardest problems in information theory—in real time.

Most people never realize this.
```

### Rules
- Short lines (2-7 words)
- One idea per tweet
- White space = readability = dwell time
- End tweet 3 with setup for "what's wrong with current solutions"

## TWEETS 4-5: THE BAD EXPLANATIONS

### Goal
Show why conventional wisdom fails. Build intellectual credibility.

### Format
```
## Tweet 4: Common Advice
[What people usually say]

[Why it seems reasonable]

[But watch what happens...]

## Tweet 5: Why It Fails
[The fatal flaw]

[The test that reveals it's "easy to vary"]

[Cliffhanger to good explanation]
```

### Example
```
## Tweet 4
Most advice is useless:

"Know your audience" → symptom, not solution
"Use analogies" → how?
"Practice more" → practice WHAT?

These describe the problem.

They don't solve it.

## Tweet 5
Notice: you can swap any of these for each other.

"Practice more" → "Focus more" → "Try harder"

Same advice, different words.

That's the sign of a bad explanation: easy to vary.

Here's what actually works...
```

### The Easy-to-Vary Test
Show that the common explanation can be varied without changing its "truth":
- Swap key terms → still "works"
- Change examples → still "fits"
- That means it explains everything = explains nothing

## TWEET 6: THE CLIFFHANGER

### Purpose
Prevent drop-off before the good stuff. Create micro-tension.

### Cliffhanger Templates
- "But here's what actually happens..."
- "This is where everyone gets it wrong:"
- "Most people stop here. Don't."
- "The real mechanism is counterintuitive..."
- "But watch what happens when you test it:"

### Example
```
## Tweet 6
Here's the real framework:

Articulation is COMPRESSION.

You're encoding high-dimensional understanding into low-bandwidth language.

Like compressing a file—except the receiver has a different operating system.

Let me explain...
```

## TWEETS 7-10: THE GOOD EXPLANATION

### Goal
Deliver the core insight with mechanism, evidence, and reach.

### Structure
- **Tweet 7**: The mechanism (why it works)
- **Tweet 8**: Concrete example #1 (famous/historical)
- **Tweet 9**: Concrete example #2 (technical/specific)
- **Tweet 10**: Why this explains more than expected (reach)

### The Specificity Rule
| Weak | Strong |
|------|--------|
| "Great speakers" | "Lincoln" |
| "Short speech" | "272 words" |
| "Technical formula" | "E=mc² (5 characters)" |
| "Encoding mismatch" | "Speaking JPEG to someone who reads pixels" |

### Example
```
## Tweet 7
This explains everything:

Why experts struggle to teach beginners
Why your first draft is garbage
Why some tweets hit and others flop

It's not about intelligence.

It's about compression strategy.

## Tweet 8
The Gettysburg Address: 272 words.

More impact than 10,000-word speeches from that era.

Lincoln didn't explain everything.

He found the compression key: "All men are created equal" as a proposition tested by blood.

Everything else? Reconstructable.

## Tweet 9
E=mc²

5 characters.

Encodes years of Einstein's work.

But here's the catch:

It only works if you have the decompression algorithm.

Maximum compression requires matched mental models.

## Tweet 10
This is why expert-to-novice communication breaks:

Expert sees: "weak king safety"
Novice sees: "knight on e4"

Different encoding formats.

The expert isn't bad at explaining.

They're speaking JPEG to someone who only reads pixels.
```

## TWEETS 11-12: THE PAYOFF

### Goal
Make it actionable. Create "saveable" content.

### What Makes Content Saveable
- Numbered frameworks (4 steps to X)
- Counterintuitive rules
- Quotable one-liners
- Visual patterns (→, =, vs)

### Format
```
## Tweet 11: The Framework
[Numbered steps]

[Each step is specific, actionable]

[Can be applied immediately]

## Tweet 12: The Quick Win
[One thing they can try NOW]

[Takes 2 minutes]

[Proves the framework works]
```

### Example
```
## Tweet 11
4 steps to compress better:

1. Identify the ONE load-bearing insight
2. Remove everything else
3. Add exactly ONE example
4. Test: Can they rebuild the rest?

If not, your compression key is wrong.

## Tweet 12
The 2-minute trick that saves 20:

Before explaining anything complex, write the ONE sentence that makes everything else obvious.

Can't find it?

You haven't found your compression key yet.

Don't start talking until you have it.
```

### Format
```
"But [strongest objection]!"

[Acknowledge why it seems valid]

[Reframe or explain why it actually supports your point]
```

```
More shared context = higher compression ratio.

This is WHY jargon exists.

It's efficient—when both sides have the decompression key.
```

## TWEETS 15-16: THE CTA

### The 75x Reply Strategy
Remember: You replying to someone's reply = 75x boost.

Design your CTA to generate replies you can respond to.

### CTA Formula
```
[Challenge/Action] + [Save Prompt] + [Reply Prompt] + [Follow Hook]
```

### Example
```
## Tweet 15
Try this now:

Take something you "can't explain well."

Write 200 words.
Cut to 100.
Cut to 50.

Notice what survives.

That's your compression algorithm revealing itself.

## Tweet 16 (CTA)
If this reframed how you think about communication:

→ Save this thread (you'll need it)
→ Reply with what YOU struggle to explain
→ Follow for more frameworks that actually work

The difficulty doesn't disappear.

But now you know what you're solving.
```

### CTA Checklist
- [ ] Actionable challenge
- [ ] Save/bookmark prompt (for savers)
- [ ] Reply prompt with specific question (for engagers)
- [ ] Follow prompt with reason (for growth)
- [ ] Callback to thread's core insight

## Formatting Rules

### Per-Tweet Formatting
```
[Short line 2-7 words]

[Another short line]

[Third line if needed]

[Never more than 4 lines without break]
```

### Character Limits
- **Target**: Under 250 characters (leaves room for RT)
- **Hard max**: 280 characters
- **Mobile-first**: Short lines, lots of breaks

### Visual Elements
- **Arrows**: → for steps/implications
- **Bullets**: • for lists
- **Equals**: = for definitions
- **Numbers**: 1, 2, 3 for sequences
- **Emojis**: Max 1-2 per thread (🧵 and one other)

## What NOT to Do

### Thread Killers
- Dense paragraphs (walls of text)
- Vague hooks ("Here are some thoughts...")
- No stakes ("Thread about X")
- Skippable tweets (doesn't advance or reveal)
- Weak CTA (no engagement prompt)
- Over-emoji'd (looks unprofessional)
- Hashtag spam in body (save for reply)
- Preachy tone ("You must...", "Obviously...")

### The Ultimate Test
Before posting, ask:
1. Would I stop scrolling for this?
2. Would I read all 15 tweets?
3. Would I save this?
4. Would I reply to the CTA?
5. Would I follow for more?

If any answer is "no," revise.
